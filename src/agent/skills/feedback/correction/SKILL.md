# 纠错反馈

## 触发条件

当 `deviation_score < 0.3` 时触发纠错反馈。

## 反馈目标

1. 指出回答中的错误
2. 提供正确的理解/做法
3. 给出明确的改进方向
4. 保持专业和鼓励的态度

## 纠错模板

### 结构

```
1. 肯定尝试 (缓和语气)
2. 指出问题 (具体明确)
3. 解释原因 (帮助理解)
4. 给出建议 (明确方向)
```

### 示例

```
"感谢你的回答，但我注意到你对缓存的理解有一些偏差。

具体来说，你提到'使用 Redis 做持久化存储'其实是不准确的。Redis 是内存数据库，持久化需要配合 RDB 或 AOF。

正确的做法是：如果需要持久化，应该... 

建议你在学习 Redis 时，重点关注以下区别：
1. 内存存储 vs 持久化存储
2. 缓存策略 (LRU/LFU) vs 数据存储
3. Redis 适用场景 vs 数据库适用场景"
```

## 语气规范

| ✅ 推荐 | ❌ 避免 |
|--------|--------|
| "你的理解有一点偏差" | "你答错了" |
| "这里需要注意" | "这是错的" |
| "建议了解" | "你不懂" |
| "正确做法是" | "你应该" |

## 纠错与追问的关系

纠错后根据情况决定：
- 重新提问类似问题
- 直接给出正确答案
- 跳过该话题
