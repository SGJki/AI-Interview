# 点评反馈

## 触发条件

当 `deviation_score >= 0.6` 时触发点评反馈。

## 反馈目标

1. 明确肯定优秀表现
2. 指出做得好的具体点
3. 适度鼓励继续
4. 可以自然转向下一话题

## 点评模板

### 结构

```
1. 总体评价 (简短有力)
2. 具体优点 (有理有据)
3. 适当时机转向 (自然衔接)
```

### 示例

```
"回答得非常出色！

特别是你对 Redis 缓存问题的分析很全面，不仅提到了常见策略，还结合了项目实践说明。

可以看出你对这块有深入的理解和实战经验。"

# 自然转向
"既然你提到性能优化，能再说说你项目中还有哪些优化手段吗？"
```

## 点评要点

### ✅ 应该包含
- 具体的优点描述
- 与问题相关的评价
- 真诚的鼓励

### ❌ 避免
- 过于笼统的"很好"
- 空泛的"不错"
- 过度夸大的言辞

## 分数细化评价

| 分数 | 评价 | 转向建议 |
|------|------|----------|
| 0.9 - 1.0 | "完美" | 深入/新话题 |
| 0.8 - 0.9 | "优秀" | 继续深入 |
| 0.7 - 0.8 | "良好" | 自然转向 |
| 0.6 - 0.7 | "不错" | 推进流程 |
