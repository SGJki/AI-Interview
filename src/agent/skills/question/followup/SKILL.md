# 追问生成策略

## 追问决策

```
deviation_score (回答质量)
    │
    ├── >= 0.8 (优秀) ──→ 继续深入 / 转到新话题
    │
    ├── >= 0.6 (良好) ──→ 肯定 + 追问细节
    │
    ├── >= 0.3 (一般) ──→ 引导补充
    │
    └── < 0.3 (较差) ──→ 纠错 + 重新解释
```

## 追问方向

### dev < 0.3: 深入技术细节
```
追问方向: "请具体说明你是如何实现的"
关注点: 技术实现、具体做法
```

### 0.3 <= dev < 0.6: 引导扩展
```
追问方向: "能否补充一些项目背景"
关注点: 场景、约束、团队协作
```

### dev >= 0.6: 肯定 + 探索
```
追问方向: "很好，你在这个过程中遇到最大挑战是什么？"
关注点: 挑战、反思、深度思考
```

## 追问模板

### 技术细节追问
```
能详细说说你是如何{implementation}的吗？
你提到了{technology}，具体用在了哪些场景？
这个方案相比其他选择有什么优势？
```

### 场景补充追问
```
能补充一下项目的背景和目标吗？
你的团队规模是怎样的？你负责哪部分？
这个项目有什么特别的挑战？
```

### 深度探索追问
```
你在项目中遇到最大挑战是什么？如何解决的？
如果重新来做，你会如何改进？
这个经验对你后续工作有什么影响？
```

## 追问终止条件

满足以下任一条件终止追问：

1. `followup_depth >= max_followup_depth`
2. `deviation_score >= 0.9` (非常优秀)
3. `len(followup_chain) >= 5` (追问链过长)
4. 用户请求结束

## 追问链记录

每次追问记录：

```python
followup_chain = [
    "q_initial_1",      # 初始问题
    "q_followup_1",     # 第一次追问
    "q_followup_2",     # 第二次追问
]
```
