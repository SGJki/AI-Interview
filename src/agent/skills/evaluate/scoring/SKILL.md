# 评分方法

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术准确性 | 40% | 回答的技术内容是否正确 |
| 完整性 | 25% | 是否完整回答了问题 |
| 深度 | 20% | 是否有深度见解 |
| 表达清晰度 | 15% | 表述是否清晰有条理 |

## 评分等级

| 等级 | deviation_score | 说明 |
|------|-----------------|------|
| 优秀 | 0.8 - 1.0 | 完整、准确、有深度 |
| 良好 | 0.6 - 0.8 | 基本准确，稍有欠缺 |
| 一般 | 0.3 - 0.6 | 部分准确，需要引导 |
| 较差 | 0.0 - 0.3 | 偏离主题或严重错误 |

## 评估标准

### 技术准确性检查

```
✅ 正确: "使用 Redis 做缓存，设置了 5 分钟过期时间"
❌ 错误: "用 Redis 做持久化存储"

⚠️ 部分正确: "用索引优化查询" (未说明索引类型)
```

### 完整性检查

```
✅ 完整: "我们采用微服务架构，使用 Docker 容器化，K8s 编排"
❌ 不完整: "我们用微服务"

⚠️ 需要追问: "我用过微服务" (缺少细节)
```

### 深度检查

```
✅ 有深度: "缓存雪崩通过多级过期 + 随机过期 + 熔断降级解决"
❌ 无深度: "加了缓存"

⚠️ 表面: "查询变快了" (缺少具体数据)
```

## 评估结果格式

```python
{
    "deviation_score": 0.75,  # 0.0 - 1.0
    "is_correct": True,        # 是否符合标准
    "key_points": [            # 关键得分点
        "提到了索引优化",
        "说明了具体场景"
    ],
    "missing_points": [        # 缺失点
        "未说明索引类型",
        "未提到查询优化细节"
    ],
    "suggestions": [            # 改进建议
        "可以补充 B+树索引的原理",
        "可以说明优化效果的具体数据"
    ]
}
```
